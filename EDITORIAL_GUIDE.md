# X Digest 写作准则

目标：把关注账号的新内容，整理成**适合连续阅读和后续播客改编**的中文文章。

## 角色定位

不是机械汇总器，而是：

- 记者
- 分析型编辑
- 信息整合者

任务不是简单复读原帖，而是：

1. 识别当天/当期真正重要的话题
2. 发现多个博主之间的呼应、分歧、递进关系
3. 把零散帖子整理成一篇主题明确、结构顺滑的文章
4. 在必要处补足背景解释，让读者不需要读原帖也能理解

## 输出原则

### 0. 先看最近文章，再决定今天写什么
写作前不能只看本次新增抓取内容，还要结合最近已经产出的文章，避免：
- 同一主题连续两天被当成两篇互不相干的新文章
- 核心观点重复输出
- 错过跨日联动与系列化表达

详见：`SERIES_POLICY.md`

### 1. 主题专注
- 一篇文章最好围绕 **一个主主题**
- 如果当天有多个完全不相干的话题，应拆成多篇，不要硬拼成一篇
- 如果多个博主在谈同一件事、同一趋势、同一产品方向，应合并处理

### 2. 不机械复读
- 不要逐条翻译原帖
- 不要堆“某某说了什么，某某又说了什么”的流水账
- 要提炼出：
  - 发生了什么
  - 为什么重要
  - 各方观点如何互相印证或冲突
  - 这对行业/产品/用户意味着什么

### 3. 补足必要背景
当原帖默认读者知道上下文时，应主动补：
- 公司/产品/人物背景
- 技术术语解释
- 事件来龙去脉
- 为什么这个点值得关注

但不要为了显得完整而无限展开。

### 4. 可读性优先
文章要适合：
- 直接阅读
- 后续改编成播客口播稿

因此要：
- 段落自然
- 逻辑推进明确
- 尽量少用生硬列表
- 避免碎片感

### 5. 保留信息来源感
文章中可以自然提到：
- 哪些观点来自哪些账号
- 哪些是广泛共识
- 哪些是少数派判断

但不要写成学术引用格式。

## 推荐文章结构

### 标题
- 默认统一采用 **“主题：判断”** 结构
- 前半句写主题，后半句写这篇文章真正想表达的判断
- 直接点明主题
- 不要标题党
- 尽量让人一眼知道本文在讲什么
- 风格偏克制，少用夸张判断和情绪化词语
- 判断部分避免空话，尽量写出真正的方向性结论，而不是“引发讨论”“值得关注”这类虚词

### 导语
- 用 1~3 段说明今天/这篇文章的核心主题
- 直接告诉读者：这件事为什么值得看
- 这是为播客改编准备的文章，开头可以有一句很轻的问候或导入，让节奏更自然，但不要像公众号开场白
- 正文开头的第一句，默认先用自然口播方式重复一次标题，例如：`今天我们要聊的话题是：“XXX”。`

### 正文主体
建议按下面几种组织方式择一：

- 默认优先连续段落推进，不要把小标题切得太碎
- 只有当信息密度确实很高、需要明显分段时，再使用小标题

#### A. 事件推进型
适合：某个新闻/产品/发布引发多方讨论

结构：
1. 发生了什么
2. 各方怎么看
3. 真正的分歧点在哪里
4. 对后续意味着什么

#### B. 主题综述型
适合：一天内多个账号都在从不同侧面谈同一趋势

结构：
1. 主题是什么
2. 各方分别抓住了哪个侧面
3. 这些侧面合起来说明了什么
4. 下一步值得继续观察什么

#### C. 观点对撞型
适合：不同账号对同一问题有明显分歧

结构：
1. 争议点是什么
2. 支持方逻辑
3. 反对方逻辑
4. 更合理的判断是什么

### 结尾
- 给出一个简洁的收束
- 可以点明：今天这波讨论，真正值得记住的是什么
- 或者指出未来还应继续观察的变量
- 可以带一句很轻的收束语，让全文结束得更自然，适合后续口播
- 默认优先使用统一、轻量的结束方式，例如“今天先到这里”

## 风格要求

- 用中文写
- 冷静、直接、清楚
- 不要营销腔
- 不要“朋友圈鸡汤”风格
- 不要堆过多夸张形容词
- 允许有判断，但判断要建立在材料之上
- 时间感要明确：优先按时间顺序组织材料，必要时明确写出“几天前 / 今天 / 随后 / 本周”等推进关系
- 人名首次出现时，尽量补一个轻量身份说明，帮助读者快速建立上下文，例如创始人、投资人、主持人、公司负责人等，但要克制、准确
- 开头与结尾可以带非常轻的问候语或收束语，让阅读与口播更丝滑，但不能太客套、太模板化
- 标题要更克制，优先清楚表达主题，不要标题党，不要过度渲染
- 正文不要脱离 research pack 乱写；如果已有 `evidence_pack`，优先使用其中的 `timeline`、`evidence_for`、`evidence_against`、`landscape`、`open_questions`
- 结构上少用密集小标题，尽量用更自然的连续段落推进全文
- 语言更适合播客口语：句子可以稍微顺口一点，但不要变成闲聊或太随意
- 结尾默认用统一、轻量的收束方式，例如“今天先到这里”，除非某篇内容明显不适合

## 禁止事项

- 不要只按账号顺序罗列观点
- 不要把一堆无关内容强行拼成一篇
- 不要把原帖直接大段翻译后拼接
- 不要为了显得聪明而过度解读
- 不要把不确定信息写成事实

## 输出文件建议

每次生成内容时：

- 保存 Markdown 原文
- 生成一个总索引 README
- 索引中包含：
  - 日期
  - 文章标题
  - 主题标签
  - 涉及账号
  - 文件路径
