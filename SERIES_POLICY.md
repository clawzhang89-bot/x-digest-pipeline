# 系列文章 / 去重 / 跨日联动策略

这是 x-digest 项目的高级编辑策略，用来避免机械式“只看当天新增”的问题。

## 核心目标

1. **跨日联动**：不要因为按日抓取，就割裂同一话题在相邻几天的演进
2. **避免重复成文**：如果最近已经写过同一主题，不要换个说法再写一遍
3. **支持系列化**：如果同一主题出现新材料，应以“续篇 / follow-up”方式推进，而不是每次从头写一遍

## 原则一：不要机械地只看当日增量

虽然抓取层是增量的，但写作层不能只盯着“今天新增了什么”。

写作时应至少结合：
- 本次增量抓取内容
- 最近几天已生成文章的标题、摘要、主题标签
- 必要时回看上一次同主题文章涉及的核心论点

### 目的

避免错过这种情况：
- A 博主今天抛出一个判断
- B 博主明天补充了落地细节
- C 博主后天提出反对意见

这三天其实应该被视为同一主题的连续展开。

## 原则二：先做主题去重，再决定是否成文

在生成新文章前，应先回答：

1. 这批新材料属于全新主题吗？
2. 还是最近已经写过的主题的延续？
3. 如果是延续，新增信息是否足够支持出一篇 follow-up？
4. 如果新增信息很少，就不单独成文，留待后续继续累积。

### 去重标准

若满足以下任一情况，应优先视为“同主题延续”而不是新文章：

- 核心讨论对象相同（同一公司 / 产品 / 模型 / 发布）
- 争论点相同（例如“agent 是否真正替代 app”）
- 行业判断主线相同（例如“具身智能的 bottleneck 在数据还是控制”）
- 最近文章已经覆盖了大部分背景，今天只是出现小幅补充

## 原则三：允许并鼓励系列文章

如果某个主题持续多天都有新材料，优先使用系列化表达。

### 系列文章的推荐结构

新文章里可以自然带一句：

- 上一篇我们关注的是 X
- 这次的新变化在于 Y
- 如果说上一篇讨论的是方向判断，这一篇补充的是执行细节 / 反方观点 / 市场反馈

### 系列文章的价值

- 让读者看到一个主题是如何演进的
- 降低重复解释成本
- 更适合后续播客化，因为有叙事连续性

## 建议的数据层实现

后续正式实现时，至少维护以下元数据：

### 文章元数据
每篇文章保存：
- `title`
- `date`
- `summary`
- `tags`
- `entities`（公司 / 产品 / 人物 / 技术概念）
- `series_id`（如果属于某个系列）
- `series_title`
- `is_followup`
- `source_accounts`
- `source_post_ids`

### 主题去重信号
生成新文章前，应对比最近文章：
- 标题相似度
- 摘要相似度
- 标签重合度
- 实体重合度
- 核心观点相似度

## 编辑决策建议

面对一批新材料时，先做三选一：

### A. 新主题 → 新文章
当新增材料明显在讲一个新问题、新事件、新趋势时，直接成新文。

### B. 老主题，但有明显新增 → follow-up / 系列新篇
当主题相同，但新增信息足以改变读者理解、补足关键空白或带来明显分歧时，写续篇。

### C. 老主题，新增不足 → 暂不成文
当只是重复表达、噪音增多、没有新信息时，不生成新文章。

## 写作要求

### follow-up 文章不要这样写：
- 把旧文章内容从头重写一遍
- 和上一篇高度重复，只换个标题

### follow-up 文章应该这样写：
- 用 1~2 段快速回顾前情
- 明确指出这次新增材料带来了什么新变量
- 重点写“变化”而不是“重复背景”

## 当前阶段的落地要求

在项目下一阶段接入大模型成文时，应至少做到：

1. 读取最近文章索引与摘要
2. 对新增材料做主题匹配
3. 判断：新主题 / follow-up / 暂不成文
4. 输出文章时写明：
   - 是否属于系列
   - 如果是 follow-up，对应哪一篇旧文
   - 新增信息在哪里
